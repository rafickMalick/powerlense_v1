# STATE — PowerLens

## Intégration MQTT <-> WebSocket
- `MqttModule` (powerlens-backend/src/mqtt/) reçoit les mesures, événements et ACKs depuis le broker (`powerlens/+/+/measure`, `/event`, `/ack/+`).
- `MeasurementListener` insère en base (`EnergyMeasurement`, async) puis émet via `RealtimeGateway` (`measurement`, `alert`, `circuit:status`) et déclenche `RuleEngineService.evaluateMeasurement`.
- WebSocket : broadcast global, CORS ouvert, sans auth — limitation connue (voir docs/audit-v1.md).

## Moteur de règles
- `RulesModule` (CRUD) + `RuleEngineService` (évaluation), conditions/actions JSONB sur `Rule`.
- État d'évaluation (rising-edge + cooldown 30s) en mémoire (Map), non partagé multi-instance.
- Types supportés : THRESHOLD, SCHEDULE, AND/OR, EVENT, PRESENCE (tous implémentés — la mention "PRESENCE non implémenté" ici était obsolète, corrigée en V8).
